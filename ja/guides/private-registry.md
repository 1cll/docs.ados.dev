# プライベートレジストリ対応

ADOS はエージェント実行時にプライベートパッケージレジストリの認証情報を自動的に注入します。

## 対応レジストリ

| タイプ | 生成ファイル | 認証方式 |
|--------|-------------|---------|
| **npm** | `.npmrc` | トークン / Basic Auth |
| **pip** | `pip.conf` | トークン / Basic Auth |
| **Maven** | `settings.xml` | トークン / Basic Auth |

## セットアップ

1. ダッシュボードの **Settings** → **Private Registries** に移動
2. **Add Registry** をクリック
3. レジストリ URL を入力（例: `https://npm.pkg.github.com`）
4. 認証方式（トークンまたは Basic Auth）を選択
5. 認証情報を入力
6. 必要に応じて特定のリポジトリにスコープを限定
7. **Save** をクリック

## 動作の仕組み

1. **実行前** — エージェントが Issue の処理を開始すると、ADOS はワークスペースディレクトリに適切な認証設定ファイル（`.npmrc`、`pip.conf`、`settings.xml`）を生成
2. **実行中** — エージェントは `npm install`、`pip install`、`mvn` コマンドで通常通りプライベートパッケージをインストール可能
3. **実行後** — セキュリティのため、設定ファイルはワークスペースから自動的に削除

## npm スコープレジストリ

npm では、特定の組織にレジストリをスコープできます。npm スコープ（例: `@myorg`）配下でプライベートパッケージをホストしている場合に便利です。

**例**: `https://npm.pkg.github.com` を `@myorg` にスコープして登録すると、生成される `.npmrc` は以下のようになります：

```
@myorg:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=<token>
```

これにより、エージェントはプライベートレジストリから `@myorg/private-package` をインストールしつつ、他のパッケージには公開 npm レジストリを使用できます。

## リポジトリスコープ

デフォルトでは、登録されたプライベートレジストリはすべてのリポジトリに適用されます。必要に応じて特定のリポジトリに制限できます：

1. レジストリ設定で **Scope to repositories** を有効化
2. このレジストリへのアクセスが必要なリポジトリを選択
3. **Save** をクリック

選択されたリポジトリの Issue を処理するエージェントのみがレジストリ認証情報を受け取ります。

## セキュリティ

- 認証情報は Firestore に暗号化して保存
- 認証設定ファイルはエージェント実行中のみワークスペースに書き出し
- 実行完了後（成功・失敗問わず）に設定ファイルは自動クリーンアップ
- 認証情報はリポジトリにコミットされない
