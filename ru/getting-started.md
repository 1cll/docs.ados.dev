# Быстрый старт

Начните работу с ADOS за несколько шагов. **Первая автоматизированная задача за 5 минут.**

## Предварительные требования

- Аккаунт GitHub / GitLab / Bitbucket
- Доступ на запись в целевой репозиторий

## Шаг 1: Создание аккаунта

1. Перейдите на [панель управления ADOS](https://ados-platform-dashboard.web.app)
2. Зарегистрируйтесь через **Google-аккаунт** или **GitHub-аккаунт**
3. При первом входе появится мастер настройки

> [!TIP]
> 14-дневный пробный период Pro-плана активируется автоматически. Кредитная карта не требуется.

## Шаг 2: Подключение GitHub

1. Перейдите в **Settings** → **GitHub Integration** в боковом меню
2. Нажмите **Install GitHub App**
3. На GitHub выберите организацию/репозитории для установки
4. После установки вы будете перенаправлены обратно на панель управления

Подробности см. в [Интеграция с GitHub](ru/setup/github.md).

## Шаг 3: Регистрация репозитория

1. Перейдите в **Settings** → **Repositories**
2. Нажмите **Add Repository**
3. Выберите нужный из списка подключённых репозиториев
4. Настройте следующие параметры:

| Параметр | Описание | Рекомендуемое значение |
|----------|----------|----------------------|
| **Label** | Метка задач, которые отслеживает ADOS | `ados` |
| **Target Branch** | Базовая ветка для PR | `main` |
| **Default AI Agent** | Основной ИИ-агент | `claude` |
| **Model** | Модель ИИ | `claude-sonnet-4` |

5. Нажмите **Save**

## Шаг 4: Создание задачи

Создайте задачу (Issue) в целевом репозитории и добавьте настроенную метку (например, `ados`).

```
Title: Add rate limiting to user authentication

Body:
- Add rate limiting to the /api/v1/auth/login endpoint
- Limit to a maximum of 10 login attempts per minute from the same IP
- Return 429 Too Many Requests when the limit is exceeded
- Add tests as well
```

> [!TIP]
> Чем подробнее описание задачи, тем выше качество результата ИИ.

## Шаг 5: Автоматическая обработка ИИ

ADOS автоматически обнаруживает задачи с меткой и обрабатывает их по следующему процессу:

1. **Разбор задачи** — Чтение заголовка, описания и комментариев
2. **Понимание репозитория** — Анализ кодовой базы
3. **Реализация кода** — ИИ-агент пишет код
4. **Запуск тестов** — Проверка, что существующие тесты проходят
5. **Создание PR** — Отправка реализации как Pull Request

Прогресс можно отслеживать в реальном времени на странице **Pull Requests** панели управления.

## Шаг 6: Ревью и слияние

Проверьте созданный ИИ PR и выполните слияние, если всё в порядке.

```bash
# Слияние через CLI
gh pr merge <PR_NUMBER> --squash
```
