# Запланированные задачи

ADOS может автоматически выполнять периодические задачи обслуживания с помощью cron-выражений. Определите регулярные задания, такие как обновление зависимостей, исправление lint-ошибок и сканирование безопасности -- ADOS будет создавать GitHub Issues по расписанию, а AI-агент реализует изменения и откроет Pull Request.

## Обзор

Запланированные задачи обеспечивают подход «настроил и забыл»:

1. **`ScheduledWatcher`** проверяет cron-выражения каждую минуту
2. Когда наступает время задачи, автоматически создаётся **GitHub Issue** (с меткой `scheduled`)
3. **`IssueWatcher`** обнаруживает Issue и запускает AI-агента
4. AI реализует изменения и создаёт **Pull Request**

```
┌──────────────────┐     ┌──────────────┐     ┌──────────────┐     ┌─────────┐
│ ScheduledWatcher │ ──▶ │  GitHub Issue │ ──▶ │  IssueWatcher│ ──▶ │  PR     │
│ (cron-триггер)   │     │  (scheduled)  │     │  (AI-агент)  │     │         │
└──────────────────┘     └──────────────┘     └──────────────┘     └─────────┘
```

## Поддерживаемые типы задач

| Тип | Описание | Заголовок Issue по умолчанию |
|-----|----------|------------------------------|
| `dependency-update` | Обновление зависимостей | `[Scheduled] Update dependencies` |
| `lint-fix` | Автоисправление lint-ошибок | `[Scheduled] Fix lint issues` |
| `security-scan` | Аудит безопасности | `[Scheduled] Security scan and fixes` |
| `custom` | Пользовательская задача | Определяется пользователем |

## Cron-выражения

Поддерживается стандартный пятиполевой синтаксис cron:

```
 ┌───────────── минута (0-59)
 │ ┌───────────── час (0-23)
 │ │ ┌───────────── день месяца (1-31)
 │ │ │ ┌───────────── месяц (1-12)
 │ │ │ │ ┌───────────── день недели (0-6, Вс=0)
 │ │ │ │ │
 * * * * *
```

### Распространённые расписания

| Выражение | Расписание |
|-----------|------------|
| `0 0 * * 0` | Каждое воскресенье в 00:00 |
| `0 9 * * 1` | Каждый понедельник в 09:00 |
| `0 0 1 * *` | 1-го числа каждого месяца в 00:00 |
| `0 0 * * *` | Каждый день в 00:00 |

## API-эндпоинты

### Список запланированных задач

```bash
GET /api/v1/settings/repos/{id}/scheduled-tasks
```

### Создание задачи

```bash
POST /api/v1/settings/repos/{id}/scheduled-tasks
```

```json
{
  "enabled": true,
  "task_type": "dependency-update",
  "cron_expression": "0 0 * * 0",
  "timezone": "Asia/Tokyo"
}
```

### Обновление задачи

```bash
PATCH /api/v1/settings/repos/{id}/scheduled-tasks/{taskId}
```

### Удаление задачи

```bash
DELETE /api/v1/settings/repos/{id}/scheduled-tasks/{taskId}
```

### Ручной запуск

```bash
POST /api/v1/settings/repos/{id}/scheduled-tasks/{taskId}/trigger
```

## Настройка через панель управления

Задачи также можно настроить через GUI в панели управления:

**Settings > Scheduled Tasks**

1. Перейдите на страницу настроек репозитория
2. Откройте вкладку **Scheduled Tasks**
3. Нажмите **Add Task** и выберите тип задачи
4. Задайте cron-выражение и часовой пояс
5. Сохраните

## Сценарии использования

### Еженедельное обновление зависимостей
Запланируйте `dependency-update` на каждое воскресенье вечером. К утру понедельника PR с обновлёнными пакетами будет готов к ревью.

### Ежедневное обслуживание lint
Запускайте `lint-fix` ежедневно, чтобы поддерживать чистоту кодовой базы без ручного труда.

### Ежемесячный аудит безопасности
Используйте `security-scan` 1-го числа каждого месяца для раннего обнаружения уязвимостей.

### Пользовательское обслуживание
Используйте тип `custom` с пользовательским заголовком и инструкциями для любых регулярных задач, специфичных для вашего проекта.

> [!TIP]
> Комбинируйте запланированные задачи с [режимом AutoPilot](ru/guides/autopilot.md) для полностью автономного обслуживания репозитория.
