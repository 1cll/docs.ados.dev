# Поддержка приватных реестров

ADOS автоматически внедряет учётные данные для аутентификации в приватных реестрах пакетов во время выполнения агента.

## Поддерживаемые реестры

| Тип | Генерируемый файл | Метод аутентификации |
|-----|-------------------|---------------------|
| **npm** | `.npmrc` | Токен / Basic Auth |
| **pip** | `pip.conf` | Токен / Basic Auth |
| **Maven** | `settings.xml` | Токен / Basic Auth |

## Настройка

1. Перейдите в **Settings** → **Private Registries** в панели управления
2. Нажмите **Add Registry**
3. Введите URL реестра (например, `https://npm.pkg.github.com`)
4. Выберите метод аутентификации (токен или Basic Auth)
5. Введите учётные данные
6. При необходимости ограничьте реестр определёнными репозиториями
7. Нажмите **Save**

## Как это работает

1. **Перед выполнением** — Когда агент начинает обработку задачи, ADOS генерирует соответствующие файлы конфигурации аутентификации (`.npmrc`, `pip.conf` или `settings.xml`) в рабочей директории
2. **Во время выполнения** — Агент может устанавливать приватные пакеты обычным способом с помощью команд `npm install`, `pip install` или `mvn`
3. **После выполнения** — Файлы конфигурации автоматически удаляются из рабочей директории для безопасности

## Реестры с областью видимости npm

Для npm можно ограничить реестр определённой организацией. Это полезно, когда вы размещаете приватные пакеты под npm-скоупом (например, `@myorg`).

**Пример**: Если вы зарегистрируете `https://npm.pkg.github.com` с привязкой к `@myorg`, сгенерированный `.npmrc` будет содержать:

```
@myorg:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=<token>
```

Это позволяет агенту устанавливать `@myorg/private-package` из вашего приватного реестра, используя публичный npm-реестр для всех остальных пакетов.

## Ограничение по репозиториям

По умолчанию зарегистрированный приватный реестр применяется ко всем репозиториям. При необходимости его можно ограничить определёнными репозиториями:

1. В настройках реестра включите **Scope to repositories**
2. Выберите репозитории, которым нужен доступ к этому реестру
3. Нажмите **Save**

Учётные данные реестра получат только агенты, обрабатывающие задачи из выбранных репозиториев.

## Безопасность

- Учётные данные хранятся в зашифрованном виде в Firestore
- Файлы конфигурации аутентификации записываются в рабочую директорию только во время выполнения агента
- После завершения выполнения (успешного или с ошибкой) файлы конфигурации автоматически удаляются
- Учётные данные никогда не коммитятся в репозиторий
