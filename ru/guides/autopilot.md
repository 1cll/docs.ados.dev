# Режим AutoPilot

AutoPilot — полностью автономный режим ADOS. Он непрерывно обрабатывает задачи от обнаружения до реализации и создания PR без ручного вмешательства.

## Обзор

В отличие от стандартного режима, AutoPilot:

- **Автоматически ставит в очередь** необработанные задачи
- **Обрабатывает несколько задач параллельно**
- **Расставляет приоритеты** через фильтры
- **Работает безопасно** с ограничениями частоты

## Включение AutoPilot

```yaml
repos:
  - name: my-repo
    workers:
      autopilot:
        enabled: true
        min_open_issues: 1          # Запуск при наличии хотя бы 1 задачи
        max_per_cycle: 3            # Макс. задач за цикл
        max_per_day: 20             # Макс. задач за день
        check_interval: 10m         # Интервал проверки
        cooldown: 30m               # Задержка между циклами
        priority_filter:            # Обрабатываемые приоритеты
          - high
          - medium
        category_filter:            # Обрабатываемые категории
          - bug
          - feature
        require_approval: false     # Требовать одобрение перед слиянием
```

## Описание параметров

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `enabled` | bool | `false` | Включить/выключить AutoPilot |
| `min_open_issues` | int | `1` | Мин. задач для запуска |
| `max_per_cycle` | int | `5` | Макс. задач за цикл |
| `max_per_day` | int | `50` | Макс. задач за день |
| `check_interval` | duration | `10m` | Интервал проверки |
| `cooldown` | duration | `30m` | Ожидание между циклами |
| `priority_filter` | []string | `["*"]` | Фильтр приоритетов |
| `category_filter` | []string | `["*"]` | Фильтр категорий |
| `require_approval` | bool | `false` | Требовать одобрение |
| `model` | string | `""` | Модель (пусто = автовыбор) |
| `focus` | []string | `[]` | Области фокуса бэклога |

## AutoPilot + AI-бэклог

Сочетание AutoPilot с [AI-генерацией бэклога](ru/guides/ai-backlog.md) создаёт полностью автоматический цикл улучшения:

```
┌────────────────┐     ┌──────────────┐     ┌────────────────┐
│  AI-бэклог     │ ──▶ │  Создание     │ ──▶ │  AutoPilot     │
│  (Анализ и     │     │  задач        │     │  (Авто-        │
│   предложения) │     │  (Авто)       │     │   реализация)  │
└────────────────┘     └──────────────┘     └────────────────┘
         ▲                                          │
         └──────────────────────────────────────────┘
                  Цикл непрерывного улучшения
```

## Механизмы безопасности

### Ограничение частоты
- Контроль объёма обработки через `max_per_cycle` и `max_per_day`
- Автоматическая обработка лимитов API

### Задержка
- Предотвращает чрезмерное потребление ресурсов
- Настраиваемое время ожидания через `cooldown`

### Утверждение
- При `require_approval: true` PR не будут слиты без одобрения
- Сохраняет процесс код-ревью команды

### Предотвращение дубликатов
- Не допускает повторной обработки одной и той же задачи

## Примеры использования

### Погашение технического долга
1. AI-бэклог определяет области для улучшения
2. AutoPilot автоматически обрабатывает задачи
3. Команда сосредоточена на ревью PR
4. Оставьте работать на выходные — к понедельнику техдолг сокращен вдвое

### Запуск новых проектов
1. Создайте шаблонный репозиторий
2. Массово создайте задачи для нужных функций
3. AutoPilot реализует их последовательно
4. Базовый код готов за считанные часы

> [!WARNING]
> AutoPilot мощный инструмент, но рекомендуется установить `require_approval: true`. Ревью ИИ-изменений людьми обеспечивает качество.
